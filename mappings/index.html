Cloudflare e-mail forwarding manager

Overview

The Cloudflare e-mail forwarding manager handles multi-destination email forwarding for your domains. It keeps a list of mappings (one source address → one or more destinations) and pushes them to Cloudflare Email Routing on demand. Cloudflare is the source of truth: mappings are read live and staged in memory, then applied with Deploy — so the status you see always reflects reality.

Using the tool

  1. Connection — paste a Cloudflare API token and pick the account. The token needs Email Routing and Workers permissions. You must be connected to view or edit mappings, since they live on Cloudflare.
  2. Destinations — every address you forward to must be verified by Cloudflare first. Add an address (or import several from CSV) to send verification emails, then click the link in each inbox. Verified addresses show a green verified badge.
  3. Mappings — add a source address (its domain must be one of your Cloudflare zones) and one or more destinations with + New mapping, or import in bulk from CSV. A blank local part creates a catch-all for that domain.
  4. Deploy — click Deploy to Cloudflare in the top bar. The wizard validates each mapping, shows what will be created or updated, and only deploys mappings whose destinations are all verified.

Cloudflare connection

Connect your Cloudflare account so the app can read live status and deploy your mappings. Your API token is stored encrypted in the local database and never leaves your machine except in calls to Cloudflare.

How do I get a token?

The token lets this app create the Email Routing rules and the fan-out Worker for you. It is stored encrypted on your machine and never leaves it except in calls to Cloudflare.

  1. Open the API Tokens page (Cloudflare dashboard → My Profile → API Tokens).
  2. Click Create Token → Create Custom Token (“Get started”).
  3. Give it a name, e.g. mappings-manager.
  4. Add these Permissions:
    • Account → Workers Scripts → Edit
    • Account → Email Routing Addresses → Edit
    • Zone → Email Routing Rules → Edit
    • Zone → Zone → Read
  5. Under Account Resources pick your account, and under Zone Resources choose All zones (or the specific domains you’ll manage).
  6. Click Continue to summary → Create Token.
  7. Copy the token shown and paste it above, then Save token.

Cloudflare shows the token value only once — if you lose it, just create another. You can revoke it any time from the same page.
